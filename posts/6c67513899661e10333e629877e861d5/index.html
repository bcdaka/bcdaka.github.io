SQL server数据库日志文件收缩

收缩数据库日志文件不需要暂停数据库服务，收缩数据库日志文件时，数据库的数据文件和日志文件会重新组织，这会影响到数据库的读写性能，导致读写速度变慢，甚至出现服务器的假死情况。因此，在执行收缩操作时，需要提前做好规划，尽量避免在高峰期进行，同时也要做好应急预案。

1、首先打开数据库，选择需要收缩日志文件的数据库右键选择属性点击选项将恢复模式完整改为简单点击确定。

2、右击数据库选择收缩文件

3、选择收缩日志，更改想要收缩到的大小点击确定

4、收缩完成后将恢复模式改回完整

此时数据库日志文件将收缩到规定大小

收缩命令：

-- 将数据库恢复模式更改为简单

ALTER DATABASE [ZiDanDataBase]  
SET RECOVERY SIMPLE;  
GO 
-- 收缩日志到1G
DBCC SHRINKFILE (N'ZhiDan_log', 1024);
GO
-- 将数据库恢复模式更改为完整
ALTER DATABASE [ZiDanDataBase]  
SET RECOVERY FULL;  
GO
