Mac 中修改默认 的Python 版本

电脑型号：MacBook Pro【芯片：Apple M2 Max.    macOS：Ventura 13.3】

一、在 Mac上删除多余的 Python 版本

解决方案：

  1、确定已安装的 Python 版本

      查看系统当前版本命令：python —version

      列出已安装的所有 Python 版本：ls /usr/bin/python*

  2、删除 Python 文件

       查找 Python 路径：which python

       删除特定的 Python 版本：sudo rm -rf <Python 文件路径>

  3、删除 Python 环境变量

       在终端运行该命令编辑环境变量配置文件：open ～/.zshrc【注意：Mac 新系统的 shell 是 zsh而不是 bash】

       在打开的文件中，删除与删除的 Python 版本有关的环境变量设置。

  4、验证删除

       运行命令：python + 版本号，如果提示zsh: command not found，说明该 Python 版本已经成功删除

二、解决 Mac 上 pyenv无法切换 Python 版本问题

   1、添加 pyenv环境

       安装pyenv：brew update

                        brew install peen

       查看 pyenv是否正确安装：pyenv —version

       查看系统安装的所有 Python 版本：pyenv versions

       安装特定的 Python 版本：pyenv install <python版本号：如 3.8.10>

       设置系统中默认的 Python 版本：pyenv global <python版本号>

   2、pyenv global 不生效

        可能是因为 path 路径没有正确配置，需要在～/.zshrc文件中添加以下内容：

            export PYENV_ROOT=\"$HOME/.pyenv\"

            export PATH=\"$PYENV_ROOT/bin:$PATH\"

            eval \"$(pyenv init —path)\"

        然后保存关闭该文件，接下来重新运行”pyenv global <Python版本号>”来全局设置 Python 版本.

请注意，如果您的Python版本已经安装在系统Python中，则需要将其从PATH中删除，以避免可能的冲突。

三、Mac 终端命令失效问题解决

      环境变量配置错误导致终端命令失效，提示zsh: command not found: ，添加此命令保存：export PATH="/usr/local/bin:/usr/local/sbin:~/bin:$PATH"

      方式一：打开环境变量配置文件：open ～/.zshrc直接编辑，然后 command s保存；

      方式二：vim ～/.zshrc，命令行编辑，输入 i 进入编辑，esc退出编辑，：wq保存退出
