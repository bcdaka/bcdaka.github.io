Flink 命令行提交、展示和取消作业

在Flink中，可以通过命令行提交、展示和取消作业。以下是如何使用命令行操作Flink作业的示例：

  1. 提交作业
./flink run -c com.example.MainClass /path/to/your/job.jar [jobArgs]

这里，run是提交作业的命令，-c后面跟着的是主类的全路径，接着是jar文件的路径和可选的作业参数。

  1. 展示作业
./flink list

这将列出当前Flink集群中所有运行的作业。

  1. 取消作业
./flink cancel JOB_ID

以上命令假设你在Flink的命令行工具目录下执行，并且你有正确的权限和配置。如果你的Flink集群部署在远程服务器上，你可能需要通过SSH或其他方式连接到远程服务器，然后执行上述命令。

命令行

  1. 启动本地 Flink 集群：
./bin/start-cluster.sh
  1. 停止本地 Flink 集群：
./bin/stop-cluster.sh
  1. 提交一个作业到 Flink 集群运行：
./bin/flink run examples/streaming/WordCount.jar
  1. 停止正在运行的作业：
./bin/flink cancel <job-id>
  1. 列出所有运行的作业：
./bin/flink list
  1. 列出作业的详细信息：
./bin/flink info <job-id>

7/ 列出savepoint能够恢复的作业：

./bin/flink savepoint <job-id> [savepointDirectory]
  1. 从savepoint恢复作业：
./bin/flink run -s :savepointMetaDataPath <program> [<arguments>]
  1. 更新运行中的作业的并行度：
./bin/flink update <job-id> -p <newParallelism>
  1. 从指定的jar或者类运行一个新的程序：
./bin/flink run -c <main-class> <jar-file> <arguments>

注意： 这些命令应在 Flink 安装目录的命令行终端中执行。每个命令的具体参数和用法可以通过在命令后添加 --help 来查看详细的帮助信息，例如：./bin/flink run --help。
